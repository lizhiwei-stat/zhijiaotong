## 📖 第一章:GPT-1(2018)——范式的起点

### 1.1 核心论文信息

- **标题**:《Improving Language Understanding by Generative Pre-Training》
- **作者**:Alec Radford 等(OpenAI)
- **时间**:2018 年 6 月
- **参数量**:1.17 亿
- **数据**:BooksCorpus,约 7000 本未出版书籍,5GB

### 1.2 GPT-1 要解决什么问题?

**时代背景**:2018 年时,NLP 领域的主流做法是:

- 每个任务(情感分析、翻译、问答)都要设计**专门的模型架构**
- 每个任务都需要大量**人工标注数据**
- 模型之间**不能复用**

**GPT-1 的野心**:能不能用**一个通用架构 + 一次预训练**,搞定所有下游任务?

### 1.3 核心方法:两阶段训练

#### 阶段一:无监督预训练(Unsupervised Pre-training)

- **任务**:给定前面 k 个 token,预测下一个 token(Language Modeling)
- **数学形式**:

$$L_1(U) = \sum_i \log P(u_i \mid u_{i-k}, \ldots, u_{i-1}; \Theta)$$

- **数据**:BooksCorpus(海量无标注文本)
- **架构**:12 层 Transformer Decoder

**💡 对应你的实战**:你 `hello_qwen.py` 里 `model.generate()` 做的就是这个——**给定前文,预测下一个 token**。Qwen 是这个训练方式的继承者。

#### 阶段二:有监督微调(Supervised Fine-tuning)

- 把预训练好的模型**接上一个任务特定的输出层**
- 用少量标注数据微调
- 不同任务的输入格式通过**特殊分隔符**区分

### 1.4 GPT 和 BERT 的架构差异(面试必考)

```
        ┌─────────────────────────────────────────────┐
        │           原始 Transformer (2017)            │
        │                                              │
        │    ┌──────────┐              ┌──────────┐   │
        │    │ Encoder  │──────────────▶  Decoder │   │
        │    │  (编码) │              │  (解码) │   │
        │    └──────────┘              └──────────┘   │
        └─────────────────────────────────────────────┘
                    │                        │
                    ▼                        ▼
              ┌──────────┐            ┌──────────┐
              │  BERT    │            │   GPT    │
              │ (2018)   │            │ (2018)   │
              └──────────┘            └──────────┘
              只用 Encoder            只用 Decoder
              双向注意力              单向(因果)注意力
              擅长理解任务            擅长生成任务
              (分类/NER/QA)           (写作/对话/续写)
```

**关键差异表**:

| 维度       | BERT                | GPT                         |
| ---------- | ------------------- | --------------------------- |
| 架构       | Transformer Encoder | Transformer Decoder         |
| 注意力方向 | 双向(看全文)        | 单向(只看过去)              |
| 训练任务   | Masked LM(填空)     | Next Token Prediction(续写) |
| 擅长       | 理解类任务          | 生成类任务                  |
| 代表模型   | BERT / RoBERTa      | GPT-2/3/4 / Qwen / LLaMA    |

**面试杀招回答**:

> "BERT 和 GPT 本质是 Transformer 的两种不同切片。BERT 拿走 Encoder 做理解,GPT 拿走 Decoder 做生成。后来事实证明 Decoder-Only 路线 scale 得更好,所以 LLM 时代是 GPT 家族的天下。"

### 1.5 GPT-1 的历史意义

- ✅ **开创了"预训练 + 微调"这一范式**
- ✅ 证明了 **Decoder-Only 架构在 NLP 可行**
- ✅ 为 GPT-2/3 铺平了道路
- ❌ 但此时**风头被 BERT 抢走了**——BERT 效果更好、发布稍晚、一下成为明星

## 📖 第二章:GPT-2(2019)——不再微调的野心

### 2.1 核心论文信息

- **标题**:《Language Models are Unsupervised Multitask Learners》
  - 注意这个标题本身就是论文的主要结论!
- **时间**:2019 年 2 月
- **参数量**:15 亿(比 GPT-1 大 10 倍)
- **数据**:WebText(从 Reddit 筛选的高质量网页),40GB

### 2.2 GPT-2 要证明什么?

**一句话**:只要**模型够大、数据够多、质量够高**,不用微调也能做很多任务。

GPT-1 已经证明"预训练 + 微调"可行。但微调有个问题:**每个任务还是要收集标注数据**。这有点违背"通用智能"的初衷。

**GPT-2 的主张**:为什么不让一个模型直接**零样本(Zero-Shot)**做所有任务?

### 2.3 什么是 Zero-Shot?

```
         传统方法(GPT-1 时代)
         ─────────────────────
         预训练 ▶ 微调(翻译数据) ▶ 翻译模型
         预训练 ▶ 微调(分类数据) ▶ 分类模型
         预训练 ▶ 微调(摘要数据) ▶ 摘要模型
         每个任务都要单独微调,都要标注数据

         GPT-2 方法(Zero-Shot)
         ─────────────────────
         预训练 ──┬─▶ "Translate to French: Hello" ──▶ "Bonjour"
                 ├─▶ "Classify sentiment: I love this" ─▶ "positive"
                 └─▶ "Summarize: [长文本]" ──▶ "[摘要]"
         一个模型 + Prompt 描述任务 = 完成任务
         不需要任何标注数据、不需要修改模型参数
```

### 2.4 GPT-2 的四个尺寸

GPT-2 论文做了**规模 Scaling 实验**,训了 4 个不同大小的模型:

| 名字   | 参数量   | 层数   | 隐藏维度 |
| ------ | -------- | ------ | -------- |
| Small  | 117M     | 12     | 768      |
| Medium | 345M     | 24     | 1024     |
| Large  | 762M     | 36     | 1280     |
| **XL** | **1.5B** | **48** | **1600** |

**关键发现**:模型**越大,Zero-Shot 效果越好**,**而且似乎还没饱和**。这个发现直接催生了 GPT-3 的"继续放大"决定。

### 2.5 GPT-2 的历史意义

- ✅ **首次严肃验证了"大模型 + Prompt"范式**
- ✅ 证明了**模型规模的威力**(scaling law 的前奏)
- ✅ 因为"生成能力太强可能被滥用",**OpenAI 最初拒绝开源完整模型**——这是 AI 安全讨论的标志性事件
- ⚠️ 但 Zero-Shot 效果还是比**专门微调**的模型差一截,没能完全替代微调

**💡 对应你的实战**:你在 Day 2 让 Qwen 批改代码、出题、生成教案——**这就是 GPT-2 式 Zero-Shot**。你没有给它任何标注数据,只给了一个 Prompt,它就能做。

## 📖 第三章:GPT-3(2020)——LLM 时代的起点

### 3.1 核心论文信息

- **标题**:《Language Models are Few-Shot Learners》
- **时间**:2020 年 5 月
- **参数量**:**1750 亿**(比 GPT-2 大 **116 倍**!)
- **数据**:Common Crawl + WebText + Books + Wikipedia,570GB
- **篇幅**:75 页论文——GPT 系列最长的一篇

### 3.2 GPT-3 的核心贡献:In-Context Learning

**这是整个 LLM 时代最重要的概念之一**。

**一句话解释 In-Context Learning**:

> 在 Prompt 里给模型几个**示例**,模型就能学会做新任务——**不用更新任何参数**。

#### 三种 In-Context Learning 方式

```
┌─────────────────────────────────────────────────────┐
│                    Zero-Shot                         │
│  ───────────────────────────────────────            │
│  [Prompt]                                           │
│  Translate English to French:                       │
│  cheese =>                                          │
│  ───────────────────────────────────────            │
│  [模型输出] fromage                                 │
│  ➤ 只告诉任务,不给示例                              │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│                    One-Shot                          │
│  ───────────────────────────────────────            │
│  [Prompt]                                           │
│  Translate English to French:                       │
│  sea otter => loutre de mer                         │
│  cheese =>                                          │
│  ───────────────────────────────────────            │
│  [模型输出] fromage                                 │
│  ➤ 给 1 个示例 + 新问题                              │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│                   Few-Shot                           │
│  ───────────────────────────────────────            │
│  [Prompt]                                           │
│  Translate English to French:                       │
│  sea otter => loutre de mer                         │
│  peppermint => menthe poivrée                       │
│  plush giraffe => girafe en peluche                 │
│  cheese =>                                          │
│  ───────────────────────────────────────            │
│  [模型输出] fromage                                 │
│  ➤ 给 N 个示例(通常 2-10 个)+ 新问题                │
└─────────────────────────────────────────────────────┘
```

### 3.3 In-Context Learning vs Fine-tuning

| 维度               | Fine-tuning          | In-Context Learning               |
| ------------------ | -------------------- | --------------------------------- |
| **需要训练数据**   | 成千上万条           | 0-10 条(放在 Prompt 里)           |
| **需要计算资源**   | 大量 GPU 算力        | 只需推理                          |
| **需要改模型参数** | 是                   | **否**                            |
| **部署**           | 每个任务一个微调模型 | **一个模型打天下**                |
| **时间**           | 几小时到几天         | 几秒到几分钟                      |
| **谁胜出**         | 在特定任务上通常更准 | 在**通用性 + 低成本**上压倒性胜利 |

**💡 对应你的实战**:你和 Qwen 说"请批改这段代码"——**这就是 Zero-Shot**。如果你再加一句"比如这样批改:xxx",就变成了 **One-Shot**。你已经天天在用 In-Context Learning 了。

### 3.4 涌现能力(Emergent Abilities)

GPT-3 论文最震撼的发现:**有些能力只有模型到达一定规模后才会出现**。

```
能力强度
   ▲
   │
   │                                    ╱──────────  "涌现能力"
   │                                   ╱           (算术、推理、...)
   │                                  ╱
   │                                 ╱
   │                                ╱
   │  ─────────────────────────────╱
   │                          临界规模
   │                         (约 10B 参数)
   └────────────────────────────────────────────▶ 模型规模
      小模型      中模型       大模型         超大模型
```

**典型涌现能力**:

- 三位数加减法
- 多步推理(Chain-of-Thought 的基础)
- 翻译低资源语言
- 理解代码逻辑
- 人类水平的对话

**为什么叫"涌现"**:小模型几乎为 0,模型突破某个规模后能力**突然跃升**。这不是线性增长,而是**相变(phase transition)**。

**💡 这解释了你的一个直观体验**:你用 Qwen2.5-**1.5B** 让它批改代码,它修对了但讲不清为什么。如果你用 **7B 甚至 72B**,就能讲清楚——这就是涌现能力的体现。这也是你 Week 5 微调要用 7B 而不是 1.5B 的根本原因。

### 3.5 GPT-3 到底有多大?

**数字感受**:

- 1750 亿参数 ≈ 700 GB 模型文件
- 训练一次成本估算 **约 460 万美元**(2020 年价格)
- 使用了 **45TB 原始文本**(清洗后 570GB)
- 如果把所有参数打印出来,每页 1000 个,需要 1.75 亿页 A4 纸

**对比**:你昨天跑的 Qwen2.5-**1.5B**,是 GPT-3 的 **1/116**。你跑 Qwen-**7B** 时,仍然是 GPT-3 的 **1/25**。

### 3.6 GPT-3 的历史意义

- 🏆 **开启了 LLM 时代**——所有后续的 ChatGPT、Claude、Qwen、LLaMA 都是 GPT-3 路线的延续
- 🏆 **确立了"Prompt 就是编程"范式**——你不再写代码,而是写 Prompt
- 🏆 **催生了 AI 工程新职业**:Prompt Engineer、RAG 工程师、Agent 开发者
- ⚠️ 仍有明显缺陷:幻觉、不会"说不知道"、对齐不足——**这些是 GPT-3.5 (ChatGPT) 和 RLHF 要解决的**

## 🔗 第四章:三代演进的总结对比

### 4.1 核心数据对比表

| 维度           | GPT-1       | GPT-2     | GPT-3               |
| -------------- | ----------- | --------- | ------------------- |
| **发布时间**   | 2018.6      | 2019.2    | 2020.5              |
| **参数量**     | 117M        | 1.5B      | **175B**            |
| **训练数据**   | 5GB         | 40GB      | **570GB**           |
| **层数**       | 12          | 48        | 96                  |
| **隐藏维度**   | 768         | 1600      | 12288               |
| **上下文长度** | 512         | 1024      | 2048                |
| **核心范式**   | 预训练+微调 | Zero-Shot | In-Context Learning |
| **开源程度**   | 全开源      | 部分开源  | **完全不开源**      |

### 4.2 三代的范式跃迁

```
    GPT-1                GPT-2                 GPT-3
  ──────────           ──────────            ──────────
   预训练               预训练                 预训练
     ↓                    ↓                     ↓
   + 微调            + Zero-Shot           + In-Context Learning
   (改参数)          (不改参数,             (不改参数,
                     只给任务描述)           给 Few-Shot 示例)
     ↓                    ↓                     ↓
   多个专用模型         一个通用模型           一个通用模型
                                              + 强大的涌现能力
                                              + Prompt 作为接口
```

### 4.3 为什么 Decoder-Only 最终胜出?

这是历史性问题,面试常问。**三个原因**:

1. **训练目标简单统一**:Next Token Prediction 一个任务搞定所有场景
2. **自回归天然适合生成**:人类说话/写作本来就是左到右的
3. **规模扩展性好**:scaling law 在 Decoder-Only 上最明显

而 BERT 路线(Encoder-Only)在理解任务上更强,但:

- 训练目标复杂(MLM + NSP)
- 不适合生成
- scaling 效果没 GPT 路线好

**最终结果**:2023 年后,几乎所有 LLM 都是 Decoder-Only——LLaMA、Qwen、Mistral、DeepSeek、Claude、ChatGPT 全都是。

## 🎯 第六章:面试话术模板(背下来就能用)

### 模板 1:"你能讲讲 GPT 的演进吗?"

> "GPT 系列经历了三代关键跃迁:
>
> GPT-1 在 2018 年**确立了'预训练加微调'范式**,用 1 亿参数证明 Transformer Decoder 可以做通用 NLP 任务,但每个下游任务还是要单独微调。
>
> GPT-2 在 2019 年**把模型放大 10 倍到 15 亿**,证明了一个核心观点——只要模型够大,**不用微调直接 Zero-Shot 也能完成任务**,这是从'一任务一模型'到'一模型多任务'的范式转变。
>
> GPT-3 在 2020 年**再放大 100 倍到 1750 亿**,提出 **In-Context Learning**——只需在 Prompt 里给几个示例,模型就能学会新任务,**连参数都不用改**。这催生了我们今天所有 LLM 应用的工程范式——Prompt 就是新的编程。
>
> 我现在做的'职教通'项目里,Qwen2.5 接续的就是 GPT-3 这条路线,我在项目里用到的每一个 Prompt,本质都是 GPT-3 里 In-Context Learning 的一次实例。"

### 模板 2:"GPT 和 BERT 有什么区别?"

> "它们是 Transformer 在 2018 年同时诞生的两个分支。BERT 只用 Encoder,双向注意力,训练目标是 Masked LM——适合理解类任务。GPT 只用 Decoder,单向因果注意力,训练目标是 Next Token Prediction——适合生成类任务。
>
> 后来事实证明 Decoder-Only 路线在规模扩展上**显著优于** Encoder-Only。BERT 在 Base 和 Large 规模上还能打,但到了百亿参数级别就被 GPT 系列远远甩开。**2023 年后主流 LLM 几乎全部是 Decoder-Only 架构**,LLaMA、Qwen、Mistral、Claude 无一例外。"

### 模板 3:"什么是 In-Context Learning?"

> "In-Context Learning 是 GPT-3 论文提出的核心范式,指模型**不通过更新参数,而通过 Prompt 中的示例来学会任务**。
>
> 分三种:**Zero-Shot** 不给示例只描述任务,**One-Shot** 给 1 个示例,**Few-Shot** 给多个示例。
>
> 和 Fine-tuning 的根本区别在于——Fine-tuning 会永久改变模型参数,每个任务需要一个专用模型;而 In-Context Learning **不改动模型**,所有任务共用一个通用模型,只是换不同 Prompt。
>
> 这是 LLM 时代的工程范式核心——**Prompt 工程替代了大量传统的微调工作**,极大降低了 AI 应用的开发门槛。"

## 📝 第七章:今天学到的 5 个关键词(面试速记)

**如果时间紧只记这 5 个,其他全忘都没事**:

1. **Decoder-Only**:GPT 系列的架构选择,只用 Transformer 的 Decoder 部分
2. **Next Token Prediction**:GPT 的训练目标,给前文预测下一个 token
3. **Zero-Shot / Few-Shot**:不给示例 / 给少量示例就能做新任务
4. **In-Context Learning**:不更新参数,仅通过 Prompt 学习新任务
5. **涌现能力(Emergent Abilities)**:模型规模突破临界点后突然出现的新能力
