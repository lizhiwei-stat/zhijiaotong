## 实验 1: 代码批改任务

### Prompt
系统: 你是一位资深的 Python 编程老师,回答简洁准确,善于用例子说明。
用户: 请批改这段 Python 代码,指出错误并给出正确版本:
def sum_list(lst):
total = 0
for i in range(len(lst) - 1):
total += lst[i]
return total
### Qwen 的回答
这段代码基本上是正确的,但有几个地方可以优化和修正:

1. `range(len(lst)-1)` 的范围应该是从 0 到 len(lst) - 2,因为最后一个元素在循环中不会被处理到。
2. 在 `for` 循环内部,应该使用 `lst[i]` 来访问列表中的元素。

以下是改进后的代码:

```python
def sum_list(lst):
    total = 0
    for i in range(len(lst)):
        total += lst[i]
    return total
```

这个版本的代码更清晰,并且能够正确计算列表的所有元素之和。

### 性能
- 生成用时: 26.5 秒
- 生成 token 数: 131
- 速度: 4.94 token/秒

### 🔍 关键观察(面试素材!)

**模型的代码修复是对的,但解释有严重问题:**

1. ✅ **代码层面**: `range(len(lst))` 确实是正确修复
2. ⚠️ **解释层面 · 第 1 点**: "最后一个元素在循环中不会被处理到" —— 描述现象正确,但没说清"为什么"(没讲清 range 半开区间的语义)
3. ❌ **解释层面 · 第 2 点**: "应该使用 lst[i] 来访问列表中的元素" —— **完全是废话**,原代码本来就是 lst[i]

### 💡 对项目的启示

这揭示了 1.5B 小模型在代码批改任务上的**典型缺陷**:
- 能凭模式匹配"修对 bug"
- 但无法给出**教学级别**的精确解释
- 容易出现"正确结论 + 错误/冗余理由"的组合

**这正是我们项目要做 7B 微调的核心动机之一**:
- 职业院校学生需要的不是"改对代码",而是"理解为什么错"
- 需要教师级别的解释能力,1.5B 达不到,7B 微调后有希望达到
